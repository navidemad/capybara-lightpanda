What works, what doesn't

Lightpanda is a headless browser by design — no layout, no paint, no compositor. Here's the honest matrix for system-test authors.

  • ✓Visit, click, fill, submit, navigate, history
  • ✓CSS & XPath selectors (XPath polyfilled)
  • ✓Cookies, headers, redirects, network
  • ✓Turbo Drive, Turbo Frames, Stimulus
  • ✓execute_script, evaluate_script
  • ✓iframes, frame switching
  • ✓JS dialogs (accept_modal :alert, dismiss_modal)
  • ✕Real screenshots (no compositor)
  • ✕Visual regression / pixel tests
  • ✕Scroll, resize, full getComputedStyle
  • ✕File uploads (input[type=file])
  • ✕Service Workers, Web Workers
  • ✕WebAuthn / passkeys
  • ✕Anything that needs layout to be correct

Need pixels? Keep cuprite for visual specs and let lightpanda own the rest. You can run both side by side — only the driver changes.
